## Prerequisites

- AntiMicroX installed ([see installation guide](/installation))
- A gamepad or joystick connected to your computer

## Create your first mapping

<Steps>
  <Step title="Launch AntiMicroX">
    Start AntiMicroX from your application launcher, Start menu, or terminal:

    ```bash
    antimicrox
    ```

    The main window displays a visual diagram of your controller. If AntiMicroX detects your gamepad, its name appears in the tab at the top of the window.

    <Note>
      AntiMicroX sets `QuitOnLastWindowClosed` to false, so closing the main window does not exit the application — it continues running in the system tray. To fully quit, right-click the tray icon and select **Quit**, or use **File > Quit** from the menu bar.
    </Note>
  </Step>

  <Step title="Identify your controller in the UI">
    When your controller is connected, a tab appears at the top of the window with the controller's name (for example, "Xbox 360 Controller" or "DualShock 4").

    If no controller tab appears:
    - Make sure your controller is connected before launching AntiMicroX
    - Click **Options → Update Joysticks** to rescan for connected devices
    - On Linux, verify the device is recognized: `ls /dev/input/js*`
  </Step>

  <Step title="Click a button on the controller diagram">
    The controller diagram shows every input on your gamepad: face buttons, shoulder buttons, triggers, thumbsticks, and D-pad.

    Click any button on the diagram — for example, the **A** button (or **Cross** on PlayStation controllers). A mapping dialog opens for that input.

    You can also press the physical button on your controller to highlight it in the diagram, then double-click it to open the mapping dialog.
  </Step>

  <Step title="Assign a keyboard key or mouse action">
    In the mapping dialog, choose what this input should do:

    - **Keyboard key** — Click **Keyboard** and press the key you want to assign. For example, press `Space` to map this button to the spacebar.
    - **Mouse button** — Click **Mouse Button** and select left, right, or middle click.
    - **Mouse movement** — Click **Mouse** and choose a direction (up, down, left, right) with a configurable speed.
    - **Script or executable** — Click **Action** and enter a command to run when the button is pressed.
    - **Macro** — Click **Macro** to record a sequence of keystrokes with optional timing delays.

    Click **OK** to save the assignment. The button on the diagram now shows a label with its mapped action.
  </Step>

  <Step title="Test the mapping">
    With the mapping dialog closed, focus any application window (a text editor works well) and press the physical button on your controller. You should see the mapped key or mouse action trigger.

    If the mapping does not work:
    - On Linux with Wayland, ensure the uinput module is loaded: `sudo modprobe uinput`
    - Make sure AntiMicroX's window is not in focus when testing — some applications ignore input while AntiMicroX is active

    <Tip>
      To verify AntiMicroX is reading raw input correctly, open **Options → Properties** to view the Joystick Status window, which shows live axis and button values from your controller.
    </Tip>
  </Step>

  <Step title="Save as a profile">
    Mappings are not automatically saved to a named file. To save your configuration as a reusable profile:

    1. Click **File > Save Profile As**
    2. Enter a descriptive name (for example, `platformer-wasd` or `fps-mouse-look`)
    3. Click **Save**

    The profile is saved as an `.amgp` XML file. You can load it again later via **File > Load Profile**, or assign it to launch automatically when a specific application is in focus using [auto profiles](/guides/auto-profiles).
  </Step>
</Steps>

## Common mapping patterns for gaming

<Tip>
  **WASD movement** — Map the left analog stick (or D-pad) axes to `W`, `A`, `S`, `D`. Set a small dead zone (5–10%) to avoid accidental drift. Use the **Axis to Button** mode in the axis mapping dialog.
</Tip>

<Tip>
  **Mouse look** — Map the right analog stick to mouse movement. In the axis mapping dialog, select **Mouse (Horizontal)** for the X axis and **Mouse (Vertical)** for the Y axis. Adjust sensitivity to match your preferred feel.
</Tip>

<Tip>
  **Quick macro for abilities** — Create a macro that sends `1`, waits 50 ms, then sends `Left Click` to trigger an ability and confirm targeting in one button press. Use the Macro editor's delay feature to add timing between inputs.
</Tip>
